Css

To prevent visual leaps, add display:none; to items, and a minimum height on item's container

.itemsContainer{
    min-height: 1000px; /* aprox ~ editable (min height) */
}

.itemDemoChange {
    display: none;
}

Html


Controls

Two attributes. data-ccc-view-name contains the name of view, the value saved in cookie.
For each view needed data-ccc-item-view, containing the css classes of each item to change.

The controls can't be eliminated (performance) but can be hidden by css (display: none;)

<button type="button" 
    data-ccc-view-name="grid" 
    data-ccc-item-view="col-lg-3 col-md-4 col-xs-6 gridClass itemDemoChange"
    >Grid</button>

<button type="button" 
    data-ccc-view-name="list" 
    data-ccc-item-view="col-lg-12 col-md-12 col-xs-12 listClass itemDemoChange"
    >List</button>

<!-- optional for clear cookie -->
<button type="button" 
    onclick="Cookies.remove('cookie-name');"
    >Clear Cookie</button>

Items

Only required a container and items with a class (important).

To change the appearance of the items you edit the CSS according to the classes of items.

<div class="row itemsContainer">
    <!-- itemDemoChange class is important for plugin init -->
    <div class="itemDemoChange">
        ...
    </div>
</div>

API

Options

Name Type value Default Help Required!
activeDefault string null Default view active on load page, one of data-ccc-view-name Required!
itemToChangeSelector string null Item common class. Example:.itemDemoChange Required!
nameCookie string null Unique value for each change view system. Required!
classActiveButton string 'active' Class of view active into control element
mobileView string null View active in mobile, one of data-ccc-view-name. If null, not change view in mobile.
mobileWidth number 480 Media query width of mobile version.
cookieDays number 5 Cookie days duration.
hideControlsOnMobile boolean false Hide views controls on enter mobile size.
effect boolean true Change view transition effect (fade).
timeEffect number 300 Effect duration. false

Methods

Method to change a view.

//normal init
var options = {
    nameCookie : 'name-cookie',
    activeDefault : 'grid',
    itemToChangeSelector : '.itemDemoChange'
};

$.fn.coolChangeCols(options);

if(theEndIsNear){
    //method to change a view (with the same options!)
    $.fn.coolChangeCols(options, 'list');
}

Events

Name Params
afterChangeView string: view
afterLoadView string: view
beforeChangeView string: view
beforeLoadView string: view
afterChangeViewMobile string: view
afterOutViewMobile string: view

Example

var options = {
    nameCookie : 'name-cookie',
    activeDefault : 'grid',
    itemToChangeSelector : '.itemDemoChange',

    //add event
    afterChangeView : function (view){
        alert(view); //string
    },
};

$.fn.coolChangeCols(options);
